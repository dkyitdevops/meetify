# AI Team Office

> Мультиагентная система для автоматизации офисных задач с использованием LLM-агентов.

---

## 1. Описание проекта

AI Team Office — это распределённая система из специализированных AI-агентов, работающих как единая команда. Каждый агент отвечает за свою область: от аналитики и разработки до коммуникаций и управления проектами.

**Цель:** Автоматизация рутинных задач, ускорение принятия решений и снижение нагрузки на человека.

---

## 2. Архитектура

```
┌─────────────────────────────────────────────────────────┐
│                    Координатор (Core)                   │
│              Распределение задач, роутинг               │
└────────────────────┬────────────────────────────────────┘
                     │
        ┌────────────┼────────────┬────────────┐
        ▼            ▼            ▼            ▼
   ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐
   │Аналитик │ │Разработчик│ │Менеджер │ │ DevOps  │
   └─────────┘ └─────────┘ └─────────┘ └─────────┘
        │            │            │            │
   ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐
   │Тестировщик│ │  HR     │ │ Дизайнер│ │Исследователь│
   └─────────┘ └─────────┘ └─────────┘ └─────────┘
```

**Принципы:**
- Каждый агент — изолированный процесс со своими инструментами
- Общение через единый шинный механизм (message bus)
- Память сессий сохраняется между запросами
- Fallback на человека при критических решениях

## 3. Список агентов

| # | Агент | Роль | Ключевые навыки |
|---|-------|------|-----------------|
| 1 | **Аналитик** | Документация, исследования | Создание спецификаций, анализ требований, написание отчётов |
| 2 | **Разработчик** | Код, архитектура | Написание кода, ревью, рефакторинг, отладка |
| 3 | **Тестировщик** | QA, автоматизация | Тест-планы, баг-репорты, автотесты |
| 4 | **DevOps** | Инфраструктура | CI/CD, деплой, мониторинг, бэкапы |
| 5 | **Менеджер** | Планирование | Таймлайны, приоритеты, отчётность, ресурсы |
| 6 | **HR** | Люди, коммуникации | Онбординг, 1:1, фидбек, команда |
| 7 | **Дизайнер** | UI/UX, визуал | Макеты, прототипы, дизайн-системы |
| 8 | **Исследователь** | R&D, инновации | Поиск решений, PoC, технологические прогнозы |

## 5. Инструкция по развёртыванию

### 5.1 Требования
- Docker 24+
- Docker Compose 2+
- 4GB RAM минимум
- OpenClaw Gateway

### 5.2 Быстрый старт

```bash
# 1. Клонировать репозиторий
git clone https://github.com/your-org/ai-team-office.git
cd ai-team-office

# 2. Настроить окружение
cp .env.example .env
# Отредактировать .env — добавить API ключи

# 3. Запустить
docker-compose up -d

# 4. Проверить статус
docker-compose ps
```

### 5.3 Проверка работы

```bash
# Отправить тестовую задачу
curl -X POST http://localhost:8080/api/v1/tasks \
  -H "Content-Type: application/json" \
  -d '{"agent": "analyst", "task": "Создать документацию"}'
```

### 5.4 Логи и отладка

```bash
# Логи всех агентов
docker-compose logs -f

# Логи конкретного агента
docker-compose logs -f analyst
```
