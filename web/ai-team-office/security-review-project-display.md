# Security Review: Отображение проектов (GitHub Issue #7)

**Дата:** 2026-03-24  
**Файл:** `/data/workspace/ai-team-office/index.html`  
**Область:** Отображение названия проекта на экранах мониторов агентов

---

## 1. XSS: Проверка escapeHtml

### Статус: ⚠️ ЧАСТИЧНО УЯЗВИМО

**Найденная функция `escapeHtml`:**
```javascript
function escapeHtml(text) {
    const div = document.createElement('div');
    div.textContent = text;
    return div.innerHTML;
}
```

**Проблемы:**
1. **Функция существует, но НЕ используется для project name** — в коде нет динамического отображения названия проекта на экранах мониторов. Все названия проектов захардкожены в HTML (статические данные).

2. **Если в будущем project name будет динамическим** — функция `escapeHtml` использует `textContent` → `innerHTML`, что является корректным методом экранирования для защиты от XSS.

**Рекомендация:**
- При добавлении динамического отображения project name — **обязательно использовать** `escapeHtml()` перед вставкой в DOM
- Рассмотреть использование `textContent` напрямую вместо `innerHTML` где возможно

---

## 2. Валидация: max 50 символов

### Статус: ❌ ОТСУТСТВУЕТ

**Проблема:**
- Нет валидации длины project name
- На экране монитора (`.screen-project`) используется фиксированный размер шрифта `9px` с `text-align: center`
- Длинные названия проектов могут:
  - Выйти за границы экрана монитора
  - Перекрыть другие элементы интерфейса
  - Исказить визуальное отображение

**CSS ограничения экрана:**
```css
.screen-content {
    position: absolute;
    top: 5px; left: 5px; right: 5px; bottom: 5px;
    /* ... */
}
.screen-project {
    font-size: 9px;
    /* Нет ограничения на длину текста */
}
```

**Рекомендации:**
1. Добавить валидацию на сервере/API: `maxLength: 50`
2. Добавить валидацию на клиенте перед отображением
3. Добавить CSS-защиту:
   ```css
   .screen-project {
       max-width: 100%;
       overflow: hidden;
       text-overflow: ellipsis;
       white-space: nowrap;
   }
   ```

## 3. Проверка на инъекции

### Статус: ⚠️ ТРЕБУЕТ ВНИМАНИЯ

**HTML-инъекции:**
- Текущий код использует статические данные — уязвимости нет
- При динамическом обновлении через `innerHTML` — риск HTML-инъекции

**JavaScript-инъекции:**
- Функция `escapeHtml` не защищает от инъекций в атрибуты (onclick, onerror и т.д.)
- Если project name вставляется в атрибуты — требуется дополнительное экранирование

**Пример уязвимости (если добавить динамику):**
```javascript
// УЯЗВИМО к XSS через атрибуты:
element.innerHTML = `<div onclick="handleProject('${projectName}')">...</div>`

// Безопасно:
element.textContent = projectName;
```
