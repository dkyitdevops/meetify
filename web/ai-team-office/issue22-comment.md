## 🔧 Анализ проблемы деплоя Issue #22

### Проблема
Файл `agents-api.js` обновлён на хосте (emoji Елены: `📚`), но изменения не попадают в Docker-контейнер.

### Причина
Docker-контейнер использует **старый образ**, в котором зашит старый файл с emoji `🧑‍🎮`.

Конфигурация:
- `docker-compose.yml` использует `context: ./server` 
- `server/Dockerfile` копирует файлы при сборке: `COPY server.js agents-api.js ./`
- Volume монтируется только для логов: `./server/logs:/var/log/ai-team-office`

**Проблема:** Файл `agents-api.js` копируется в образ при сборке (`docker build`), а не монтируется как volume. После изменения файла на хосте нужно **пересобрать образ**.

### Решение

Необходимо выполнить на целевом сервере (где установлен Docker):

```bash
cd /opt/ai-team-office

# Остановить и удалить старый контейнер
docker-compose down

# Пересобрать образ с новым файлом
docker-compose build --no-cache

# Запустить обновлённый контейнер
docker-compose up -d

# Проверить
curl http://localhost:3001/api/agents/status | grep -o 'Елена.*emoji'
# Должно быть: 📚 (не 🧑‍🎮)
```

### Скрипт для автоматического исправления

Создан скрипт `fix-deploy-issue22.sh` в репозитории. Для применения фикса на сервере:

```bash
cd /opt/ai-team-office
chmod +x fix-deploy-issue22.sh
./fix-deploy-issue22.sh
```

### Проверка текущего состояния файла

На хосте файл уже содержит правильный emoji:
```
'Елена': {
    githubUsername: null,
    role: 'КЭП ТСО',
    emoji: '📚',
    deskId: null
}
```

### Рекомендации для предотвращения подобных проблем

1. **Использовать volume для кода** (для development):
   ```yaml
   volumes:
     - ./server:/app
   ```

2. **Или автоматизировать пересборку** при изменениях:
   ```bash
   docker-compose up -d --build
   ```

3. **Добавить CI/CD pipeline** для автоматического деплоя при пуше в main
