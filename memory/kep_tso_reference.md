# Система КЭП ТСО - Справочная информация

## Общее описание
Система приёма и криптографической проверки документов с квалифицированной электронной подписью (КЭП) от территориальных сетевых организаций (ТСО). Работает в рамках оптового рынка электроэнергии под управлением АО «АТС» (Администратор торговой системы).

## Нормативная база

### Федеральный закон от 06.04.2011 № 63-ФЗ «Об электронной подписи»
Основной закон, регулирующий использование электронных подписей в РФ.

**Ключевые статьи:**
- **Статья 5:** Виды электронной подписи:
  - Простая электронная подпись (коды, пароли)
  - Усиленная неквалифицированная электронная подпись (НЭП)
  - Усиленная квалифицированная электронная подпись (УКЭП/КЭП)
- **Статья 6:** Информация в электронной форме, подписанная КЭП, признается равнозначной документу на бумажном носителе с собственноручной подписью
- **Статья 17.2:** Использование КЭП юридическими лицами
  - Представитель действует на основании доверенности
  - Доверенность должна быть включена в пакет электронных документов
  - Подписывается КЭП доверителя или нотариуса
- **Статья 17.5:** Оформление доверенностей в машиночитаемом виде (XML)

### Приказ Минцифры России от 18.08.2021 № 857
«Об утверждении единых требований к формам доверенностей, необходимых для использования квалифицированной электронной подписи»

**Требования к доверенности:**
- Формат: XML
- Подпись: усиленная квалифицированная электронная подпись (XMLDSIG)
- Срок действия приказа: до 01.03.2028

**Содержание доверенности:**
1. Наименование документа
2. Сведения о доверителе:
   - Для ЮЛ: наименование, адрес, ИНН, ОГРН, КПП, сведения о лице без доверенности
   - Для ИП: ФИО, СНИЛС, ИНН, ОГРНИП
   - Для ФЛ: ФИО, СНИЛС, ИНН
3. Сведения о представителе: ФИО, дата рождения, СНИЛС, ИНН
4. Срок действия доверенности
5. Полномочия (идентификаторы из классификатора Минцифры)
6. Право передоверия (если есть)

### Постановление Правительства РФ от 27.12.2010 № 1172
«Об утверждении Правил оптового рынка электрической энергии и мощности»

**Приложения к Положению о порядке получения статуса субъекта оптового рынка:**
- Приложение 11.1 — Требования к системам учета (СУ)
- Приложение 11.3 — Порядок установления соответствия СУ техническим требованиям
- Приложение 11.4 — Методические указания по предоставлению информации о состоянии СУ

### Приказ Минцифры России от 05.11.2025 № 1001
Изменения в Приказ № 857 (вступили в силу 01.02.2026):
- Доверителем может быть филиал/представительство
- Полное наименование ЮЛ (не просто наименование)
- Паспортные данные представителя — необязательны
- Обязательны: ФИО, дата рождения, СНИЛС, ИНН представителя
- Полномочия: идентификатор + наименование

## Сроки обработки пакетов (Приложение 11.4)

### Макет 90000 (Опросные листы)
- **120 минут** — ответная квитанция (макет 90001)

### Макеты 10000 / 20000
- **Не позднее 13:00 ближайшего рабочего дня** — если пакет подписан УКЭП сетевой организации без доверенности, или с уже подтвержденной ранее доверенностью
- **В течение 10 рабочих дней** — в остальных случаях (при наличии новой доверенности требуется дополнительная проверка)

## Требования к формату архива

### Допустимые форматы
- **Только ZIP-архивы**
- Архив должен быть совместим со встроенными средствами ОС Microsoft Windows

### Запрещённые форматы
- RAR
- 7z
- Другие форматы сжатия

### Ошибка при неверном формате
- **Код ошибки:** Keptso-21
- **Текст:** «Невозможность проверки архива документов»
- **Действие:** Пакет не принимается, направляется уведомление участнику

## Состав пакета документов

### Обязательные элементы (макеты 10000/20000)
1. Макет (XML) — макет 10000 или 20000
2. УКЭП макета — подпись XML-файла
3. При наличии представителя: МЧД + УКЭП МЧД

### Структура ZIP-архива
- Не более одного вложенного ZIP-архива с МЧД
- Не более одного вложенного ZIP-архива со сканом доверенности
- Название файла УКЭП должно совпадать с названием подписанного документа

## XML-макеты

### Макет 10000
- Данные о точках поставки/генерации
- Формат: XML
- Подробное описание в п. 5.1 технического решения

### Макет 20000
- Данные об измерительных комплексах
- Формат: XML
- Подробное описание в п. 5.2 технического решения

### Макет 90000
- Опросные листы
- Срок обработки: 120 минут

### Ответные квитанции
- **90001** — ответ на макет 90000
- **10001** — ответ на макет 10000
- **20001** — ответ на макет 20000

## Проверка машиночитаемых доверенностей (МЧД)

### Поддерживаемые форматы МЧД

| Формат | Версия | Описание |
|--------|--------|----------|
| **МЧД Версия 2** | 002 | Первая версия формата (ОКУД 0251222) |
| **МЧД Версия 3** | 003 | Актуальная версия (без ОКУД, расширенная структура) |
| **Скан** | — | Сканированная доверенность (не МЧД) |

**Примечание:** Формат `ON_EMCHD` (обменный формат МЧД) — это транспортная обёртка для МЧД версий 2 или 3. Система извлекает МЧД из ON_EMCHD и проверяет по правилам соответствующей версии.

### Основные этапы проверки МЧД
1. **Проверка в реестре МЧД ФНС** — через API Контур.Доверенность (ЦПРР ФНС)
2. **Проверка корректности подписи (УКЭП)** — валидация XMLDSIG
3. **Проверка полномочий представителя** — соответствие классификатору Минцифры
4. **Сверка ИНН/ОГРН** — доверителя и представителя
5. **Проверка срока действия доверенности**

### Система хранения МЧД
- **Локально** — доверенность хранится в реестре КЭП ТСО
- **ФНС** — доверенность найдена в реестре МЧД ФНС (ЦПРР)

При поступлении пакета система проверяет наличие МЧД в ЦПРР ФНС. Если найдена — тип устанавливается «ФНС», иначе «Локально».

### Статусы доверенностей в реестре КЭП ТСО
- **Черновик** — МЧД сохранена, но не прошла проверку корректности
- **Корректна** — доверенность прошла все проверки
- **Зависит от документа** — требуется дополнительная проверка
- **Игнорируется** — доверенность не может быть валидирована (некорректная подпись)

### Коды ошибок при проверке МЧД
- **Keptso-1** — отсутствие обязательных файлов
- **Keptso-2.5** — доверенность с таким номером уже существует в реестре
- **Keptso-8.1** — доверенность не может быть использована для подписания макетов 10000/20000
- **Keptso-8.2** — доверенность не может быть использована для подписания макетов 10000/20000 (другая причина)
- **Keptso-14.3** — отказ доверенного лица от полномочий
- **Keptso-14.4** — предоставленная МЧД отличается от МЧД в системе хранения
- **Keptso-14.5** — МЧД не применима по результатам проверки в системе хранения
- **Keptso-21** — ошибка формата архива (не ZIP)

## Интеграции

### Внешние системы
- **ИАСУ КУ** — интегрированная автоматизированная система управления коммерческого учёта АТС
- **Реестр МЧД ФНС** — для проверки машиночитаемых доверенностей
- **ASPMailer** — система обмена криптографически защищенными сообщениями

### Криптографическое обеспечение
- **КриптоПро JCP** — средство криптографической защиты информации
- **S/MIME** — формат подписи документов
- **УКЭП** — усиленная квалифицированная электронная подпись
