# Чеклист процесса разработки

## Порядок действий для каждой фичи:

### 1. 📋 Аналитик
- [ ] Изучает требования
- [ ] Создаёт GitHub Issue с описанием
- [ ] Определяет критерии приёмки

### 2. 🎨 UI-инженер (при необходимости)
- [ ] Создаёт макет интерфейса
- [ ] Описывает стили и анимации
- [ ] Сохраняет в `design.md`

### 3. 💻 Dev (Backend/Frontend/Fullstack)
- [ ] Пишет код по требованиям
- [ ] Следует макету UI-инженера
- [ ] Коммитит изменения

### 4. 🧪 QA Engineer
- [ ] Пишет тесты под новую фичу
- [ ] Запускает ВСЕ автотесты:
  - [ ] Тесты новой фичи
  - [ ] Регрессионные автотесты
  - [ ] Интеграционные автотесты
  - [ ] UI-автотесты
- [ ] Создаёт отчёт

### 5. 🔒 Security Engineer
- [ ] Анализирует код
- [ ] Проверяет уязвимости:
  - [ ] XSS
  - [ ] CSRF
  - [ ] Инъекции
  - [ ] API endpoints
- [ ] Даёт рекомендации

### 6. 🚀 DevOps Engineer
- [ ] Деплоит изменения на сервер
- [ ] Проверяет работоспособность
- [ ] Мониторит логи

### 7. ✅ Закрытие
- [ ] QA проверяет на продакшене
- [ ] Создаётся коммит в репозиторий
- [ ] GitHub Issue закрывается

---

## ⚠️ ВАЖНО:

### ❌ НЕЛЬЗЯ:
- Пропускать шаги процесса
- Деплоить без тестов QA
- Деплоить без проверки Security
- Закрывать Issue без коммита

### ✅ ОБЯЗАТЕЛЬНО:
- Каждый шаг выполняет ответственный агент
- QA запускает ВСЕ типы автотестов
- Security проверяет перед деплоем
- После успеха — коммит и закрытие Issue

| QA | agent-004 | Тесты, автотесты |
