## Приоритеты разработки

```
P0 (Critical) — Блокирует релиз, критические баги
P1 (High)     — Важные фичи, видимые пользователю
P2 (Medium)   — Улучшения UX, технический долг
P3 (Low)      — Нишевые фичи, исследования
```

## Релизный цикл

- **Patch** (v1.0.x) — багфиксы, по мере необходимости
- **Minor** (v1.x.0) — новые фичи, каждые 2-4 недели
- **Major** (vx.0.0) — breaking changes, по необходимости

## Метрики успеха

- [ ] Стабильность соединения > 99%
- [ ] Время подключения < 3 сек
- [ ] Latency < 150ms (p95)
- [ ] Поддержка до 8 участников P2P
- [ ] Поддержка до 50 участников SFU
