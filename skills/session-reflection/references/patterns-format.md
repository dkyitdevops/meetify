# memory/patterns.md — Format Guide

## Structure

```markdown
# Behavioral Patterns

*Обновлено: YYYY-MM-DD*

## Preferences
- [2026-03-19] preference: Голос для пересказа фильмов, не текст
- [2026-03-07] preference: Статус задач постфактум — не спрашивать промежуточно (кроме: схема БД, публичный API, деплой-конфиг)

## Workflow
- [2026-03-19] workflow: Cursor Agent — только echo "" | agent --trust --print, никогда nohup
- [2026-03-18] workflow: watch-pr.sh ставить сразу после пуша, не откладывать

## Communication
- [2026-03-19] communication: Краткие команды без контекста → уточнить одним вопросом, не угадывать
- [2026-03-15] communication: Предпочитает получить ответ сразу, дополнительный вопрос — в конце

## Tool
- [2026-03-19] tool: memory_search перед любым вопросом о прошлом — обязательно
- [2026-03-18] tool: iOS VPN-конфиг: только .conf файл, не vpn:// ссылка

## Decision
- [2026-03-15] decision: Cloudflare не предлагать (санкции/блокировки для РФ)
- [2026-03-17] decision: Модель по умолчанию — Kimi K2.5, Claude через claude-sub прокси
```

## Rules

1. **One line per pattern** — если нужно больше, паттерн слишком сложный → разбить
2. **Дата открытия** — всегда указывать когда паттерн впервые зафиксирован
3. **Обновлять, не дублировать** — если паттерн эволюционировал, заменить старую строку
4. **Эскалация** — паттерн, подтверждённый 2+ раза, помечать `[confirmed]`
5. **Удалять устаревшее** — если паттерн опровергнут новым поведением, удалить

## When to Escalate to MEMORY.md

Если паттерн влияет на все сессии (не только конкретную область) → добавить в MEMORY.md раздел Preferences & Habits с пометкой источника.
