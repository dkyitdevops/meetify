# Skill: kep-tso-support

## Описание
Экспертиза по системе КЭП ТСО (Квалифицированная Электронная Подпись Территориальных Сетевых Организаций) — информационной системе приёма и проверки документов с ЭП от ТСО для оптового рынка электроэнергии.

## Нормативная база

### Основные документы
1. **ФЗ № 63-ФЗ «Об электронной подписи»** (06.04.2011)
   - Статья 5: виды ЭП (простая, НЭП, КЭП)
   - Статья 17.2: использование КЭП ЮЛ и представителями
   - Статья 17.5: оформление доверенностей в машиночитаемом виде

2. **Приказ Минцифры № 857** (18.08.2021)
   - Единые требования к формам доверенностей
   - Формат: XML, подпись XMLDSIG
   - Действует до 01.03.2028

3. **Постановление № 1172** (27.12.2010)
   - Правила оптового рынка электрической энергии
   - Приложения 11.1, 11.3, 11.4

4. **Приказ Минцифры № 1001** (05.11.2025, вступил 01.02.2026)
   - Изменения в требованиях к МЧД

## Когда использовать
- Вопросы о сроках обработки документов
- Требования к форматам файлов и архивов
- Проверка состава пакета документов
- Вопросы по машиночитаемым доверенностям (МЧД)
- Коды ошибок и их расшифровка
- XML-макеты (10000, 20000, 90000)
- Интеграция с ИАСУ КУ и реестром ФНС
- Изменения в законодательстве

## Быстрые ответы

### Сроки обработки
- **Макет 90000:** 120 минут
- **Макет 10000/20000 без новой МЧД:** до 13:00 следующего рабочего дня
- **Макет 10000/20000 с новой МЧД:** 10 рабочих дней

### Формат архива
- **Только ZIP**
- RAR и другие форматы → отказ с кодом Keptso-21

### Коды ошибок
- **Keptso-21:** неверный формат архива (не ZIP)
- **Keptso-1:** отсутствие обязательных файлов
- **Keptso-2.5:** доверенность с таким номером уже существует
- **Keptso-8.1/8.2:** доверенность не может быть использована для подписания макетов
- **Keptso-14.x:** ошибки проверки МЧД в реестре ФНС

### Состав пакета
- XML-макет (10000 или 20000)
- УКЭП макета
- При наличии представителя: МЧД + УКЭП МЧД

### Требования к МЧД (с 01.02.2026)
- Формат: XML
- Подпись: КЭП доверителя
- Сведения о доверителе: полное наименование, ИНН, ОГРН, КПП
- Сведения о представителе: ФИО, дата рождения, СНИЛС, ИНН
- Полномочия: идентификатор + наименование из классификатора

### Поддерживаемые версии МЧД
- **МЧД Версия 2 (002)** — с ОКУД 0251222
- **МЧД Версия 3 (003)** — актуальная версия без ОКУД
- **ON_EMCHD** — обменный формат (транспортная обёртка для версий 2/3)
- **Скан** — сканированная доверенность (не МЧД)

### Проверка МЧД
- Проверка в реестре ФНС через API Контур.Доверенность
- Система НЕ проверяет квитанцию ФНС (Квит003) — работает напрямую с ЦПРР
- Тип хранения: «ФНС» (найдена в реестре) или «Локально»

## Примеры вопросов

**Q:** Какие версии МЧД поддерживает система?  
**A:** Поддерживаются МЧД Версия 2 (002) и МЧД Версия 3 (003). Формат ON_EMCHD — это транспортная обёртка, из которой система извлекает МЧД соответствующей версии.

**Q:** Проверяет ли система квитанцию ФНС (Квит003)?  
**A:** Нет, система проверяет МЧД напрямую в реестре ФНС через API Контур.Доверенность. Квитанция ФНС не является обязательным элементом для проверки.

**Q:** Какой срок обработки пакета с макетом 10000?  
**A:** Не позднее 13:00 ближайшего рабочего дня (если без новой МЧД) или 10 рабочих дней (если с новой МЧД).

**Q:** Можно ли отправить архив в формате RAR?  
**A:** Нет, только ZIP. RAR будет отклонён с ошибкой Keptso-21.

**Q:** Что изменилось в приказе Минцифры № 1001?  
**A:** С 01.02.2026 изменились требования к МЧД: доверителем может быть филиал, паспортные данные необязательны, обязательны СНИЛС и дата рождения представителя.

**Q:** Какие законы регулируют использование КЭП в системе?  
**A:** Основной закон — ФЗ № 63-ФЗ «Об электронной подписи». Детальные требования к МЧД — Приказ Минцифры № 857. Правила оптового рынка — Постановление № 1172.
