## Проекты

### КЭП ТСО — Система проверки электронных документов
**Дата начала:** 2026-03-06
**Статус:** Документы изучены, справочники обновлены (2026-03-13)

**Суть:** Информационная система приёма и криптографической проверки документов с КЭП от территориальных сетевых организаций (ТСО) для оптового рынка электроэнергии. Оператор — АО «АТС».

**Ключевые документы изучены:**
- Приложение 11.3 — Порядок установления соответствия систем учёта
- Приложение 11.4 — Методические указания по предоставлению информации о состоянии системы учёта
- Техническое решение КЭП ТСО (версия 1.9 от 04.02.2026)
- Руководство пользователя КЭП ТСО
- Руководство администратора КЭП ТСО
- Релизы/хотфиксы — история изменений системы
- **ФЗ № 63-ФЗ «Об электронной подписи»** — основа для КЭП и МЧД
- **Приказ Минцифры № 857** — единые требования к формам доверенностей
- **Постановление № 1172** — Правила оптового рынка электрической энергии

**Поддерживаемые форматы МЧД (уточнено 2026-03-13):**
- **МЧД Версия 2 (002)** — с ОКУД 0251222
- **МЧД Версия 3 (003)** — актуальная версия без ОКУД
- **ON_EMCHD** — обменный формат (транспортная обёртка для версий 2/3)
- **Скан** — сканированная доверенность

**Важное уточнение по проверке МЧД:**
- Система проверяет МЧД напрямую в реестре ФНС через API Контур.Доверенность
- **Квитанция ФНС (Квит003) НЕ проверяется** — это вспомогательный документ для участников
- Тип хранения МЧД: «ФНС» (найдена в ЦПРР) или «Локально" (в реестре КЭП ТСО)

**Нормативная база (расширенная):**

### 1. Федеральный закон от 06.04.2011 № 63-ФЗ «Об электронной подписи»
- **Статья 5:** Виды ЭП (простая, неквалифицированная, квалифицированная)
- **Статья 6:** Условия признания электронных документов равнозначными бумажным
- **Статья 17.2:** Использование КЭП юридическими лицами и представителями
  - Представитель действует на основании доверенности
  - Доверенность включается в пакет электронных документов
  - Подписывается КЭП доверителя или нотариуса
- **Статья 17.5:** Оформление доверенностей в машиночитаемом виде (XML)

### 2. Приказ Минцифры России от 18.08.2021 № 857
- Доверенность создается в формате XML
- Подписывается усиленной квалифицированной электронной подписью (XMLDSIG)
- Содержит сведения о доверителе (ЮЛ/ИП/ФЛ)
- Содержит сведения о представителе: ФИО, дата рождения, СНИЛС, ИНН
- Содержит полномочия (идентификаторы из классификатора Минцифры)
- Срок действия до 01.03.2028

### 3. Постановление Правительства РФ от 27.12.2010 № 1172
- «Правила оптового рынка электрической энергии и мощности»
- Приложение 11.1 — Требования к СУ (системам учета)
- Приложение 11.3 — Порядок установления соответствия СУ
- Приложение 11.4 — Методические указания по предоставлению информации

**Сроки обработки (важно помнить):**
- Макет 90000: 120 минут
- Макет 10000/20000 без новой МЧД: до 13:00 следующего рабочего дня
- Макет 10000/20000 с новой МЧД: 10 рабочих дней

**Критические правила:**
- Формат архива: **только ZIP** (RAR → отказ с кодом Keptso-21)
- МЧД проверяется в реестре ФНС через API (Контур.Доверенность)
- **Квитанция ФНС (Квит003) НЕ проверяется системой** — это документ для участников
- С 01.02.2026 новые требования к МЧД (Приказ № 1001)
- Поддерживаемые версии МЧД: 002, 003, ON_EMCHD
